# Contributing to SOSNot

## Workflow
1. **Fork & Clone**: Clone the repository to your local machine.
2. **Branching**: Create a feature branch: `git checkout -b feature/my-new-feature`.
3. **Commit Messages**: Use semantic commit messages (e.g., `feat: add land polygon drawing`).
4. **Pull Requests**: Open a PR against the `main` branch.

## Standards
- **Java/Quarkus**: Follow standard Java coding conventions. Use Jakarta EE 10 standards.
- **Frontend**: Use Angular style guide.
- **Testing**: All new features must include unit tests.

## Tools
- We use `make` for common tasks. Run `make help` to see available commands.
